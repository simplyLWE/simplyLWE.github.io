What is a Data Structure?.

What is a data structure?

Data structures are containers that hold things all of the same type.

How are are they used?

But what good is it to hold things we aren’t actually going to use in any meaningful way? That seems like a waste of space.

So we use these containers, not simply to hold things, but to also organize things, perhaps process or do something with these things in some way, and even retrieve things.

Like we have different storage containers like tupperware, moving boxes, storage organizers, backpacks, etc. There are also different data structures, or different types of containers.

Computers would be useless if all they did was store data. There are two big questions that arise with data structures.

  • How can we access individual elements?
  • How can we organize, or sort, the individual elements?

Considerations

While all data structures can be used to solve the same problem, often there are tradeoffs to consider when deciding upon which data structure to use. These tradeoffs scale between what we call time and space space complexity.

Time complexity refers to the amount of time it takes to solve the problem; while space complexity refers to the amount of storage space that is required to solve the problem.

And becuase we need to consider these factors of time and space complexity, the structure of particular data structures help us to determine the best use case for the data we are handling.

What is the structure of a data structure?

So in other words, data structures are simply a collection of things, and we have different types of data structures for different needs and purposes depending on what we plan on doing with these things, just like we have different containers to store and organize our things. We have a dresser for storing clothes. We have a tupperware for storing left over food. We have a backpack for school supplies, or hiking gear. And the structure of all these containers are all different from one another.

Though we have different data structures, they can all be organized into linear and non-linear classifications based on their structure.

For example, arrays, linked lists, stacks and queues are linear in their structure, whereas trees, hash tables, and graphs are non-linear in their structure.

These are the fundamental and popular data structures you are likely to encounter.

Linear

Elements in linear data structures are arranged one after another allow you to traverse through them in a single file line sort of fashion, whether that’s up or down, forward or backward. It’s just pretty much a straight line. For this reason, this makes them easier to implement compared to non-linear data strcutures. However, that being said, it does make the ability to access particular elements a bit more cumbersome. Since we have to traverse through the linear containers, the running cost

For example if we need to traverse through a linear container to find n, we have to traverse through each element linearly, which means if n is the last element in the container, or perhaps not in the container at all then we have a linear runtime cost, O(n).

So as we can see, as the size of the containers increase linearly, the runtime cost of the using these containers increases linearly.

Nonlinear

As we can see, they take on a hierarchical structure, which means we can cut the run time in half in generalized cases, if we know what we’re looking for. However, because of this structure, this structures renders the implemnation a bit omre difficult to implement than a linear data structure.
